<h2>v0.12.0 — Smooth scroll lock, visible inactive caret, drag-drop from Finder, dev bundle</h2>
<p>A focused round on stability + polish, plus a perf pass and a dedicated staging app for iteration.</p>
<ul>
  <li><b>Scroll while output is streaming is no longer glitchy.</b> v0.11.2's lock worked but felt jittery — the lock target was updated async from the scroll-wheel monitor, so chunks arriving between a scroll and the async update would pull the viewport back to the wrong row. The lock state is now recomputed synchronously per data chunk: lockedRow updates to current yDisp if the user is scrolled away, releases if they're back at the tail. The sync restore lands in the same render frame as the snap-to-bottom, so the bounce never reaches drawRect. Spurious lock-at-launch (scrollPosition returns 0 on fresh panes / after <code>clear</code> / alt-buffer) is also gated out — autoscroll now resumes the moment you scroll back to the bottom.</li>
  <li><b>Cursor visible in every split pane.</b> SwiftTerm draws a 3pt hollow stroke caret when its view isn't first responder, but on bright wallpapers the stroke blends with the foreground theme color and is effectively invisible. Inactive panes now also paint a brighter filled overlay (theme foreground, 55% fill + 95% border, 1.5pt) at the SwiftTerm caret's exact frame, so you can always see where the cursor lives in inactive splits. Active pane is unchanged (SwiftTerm's filled block). Overlay tracks the cursor as the shell moves it. Single-pane caret-at-launch is also fixed — a forced appearance refresh after the focus claim catches the case where SwiftTerm positioned the caret while the view's frame was still <code>.zero</code>.</li>
  <li><b>Drop files from Finder into the terminal.</b> Drag a file (or multi-select several) from Finder onto any pane — the shell-quoted path(s) get typed at the cursor, space-separated. Paths with spaces, single quotes, backticks, or shell-substitution characters are POSIX-safely single-quoted. Dropping on an inactive pane focuses it and inserts there. The pane shows a 2pt accent border on drag-enter so the target is obvious.</li>
  <li><b>Pane close X no longer sits behind the scrollbar.</b> SwiftTerm pins its NSScroller to the top edge via Auto Layout, which put the per-pane "close X" button literally on top of the scroll-up arrow. The scrollbar now starts 26pt below the pane's top (Auto Layout constraint replaced — overriding the frame in <code>layout()</code> just gets clobbered on the next pass), giving the X a clear, unambiguous hit area. The X's original discrete styling is preserved.</li>
  <li><b>Termy Dev: dedicated staging app.</b> <code>./stage.sh</code> builds a debug binary and installs it to <code>/Applications/Termy Dev.app</code> (bundle id <code>com.mees.termy.dev</code>) — separate process, separate UserDefaults domain, photo-negative dock icon so you can't confuse it with prod. Used as the iteration target for in-progress changes; prod stays untouched while you're working in a Termy session. <code>./release.sh</code> still cuts the prod release to <code>/Applications/Termy.app</code>.</li>
  <li><b>Perf pass.</b> ANSI CSI regex cached as a static (was re-compiled per output line in the trigger scan + per <code>recentVisibleText()</code> call). Per-pane idle timer 0.5s → 1.0s (halves runloop wakes with N active panes). Cwd persist debounce 0.5s → 1.0s (less UserDefaults churn on cd-heavy workflows). Git branch poll 4s → 10s (still debounced through .git/HEAD). StatusBar clock now only writes <code>@State</code> on actual minute rollover, not every 30s tick — halves StatusBar re-renders.</li>
</ul>
